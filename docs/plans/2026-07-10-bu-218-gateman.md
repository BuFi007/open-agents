## Scope decision

This slice defines the transport-neutral queue contract that BullMQ workers will consume. It deliberately does not make BullMQ the canonical graph, approval plane, wallet state machine, or agent DAG. Canonical writes, enrichment, embeddings, projections, and repair have separate worker profiles with bounded retries, DLQ limits, concurrency, and fairness flags.

## Sections 1–8

1. **Input validation — PASS.** Job ID, workspace ID, idempotency key, profile, and duplicate IDs are validated.
2. **External identifiers — PASS.** Every job has an explicit workspace and idempotency boundary; no provider defaults.
3. **Side-channel/fraud surface — PASS.** Queue payloads are opaque data; no approvals, wallet intents, or canonical writes are executed by the plan.
4. **Configuration — PASS with follow-up.** Profiles are centralized and bounded; deployment overrides must be validated and audited.
5. **Money/time/reversibility — PASS with follow-up.** Retry/backoff and DLQ bounds are explicit; job processors must keep canonical effects idempotent.
6. **Failure modes — PASS.** Unknown profiles, malformed jobs, and duplicates fail before dispatch.
7. **Observability — PASS with follow-up.** Profile metadata is suitable for lag/retry/fairness metrics; worker events still need Eve trace integration.
8. **AI output verification — PASS.** Tests prove separation of canonical writes from high-throughput embedding work and reject missing tenant/idempotency scope.
