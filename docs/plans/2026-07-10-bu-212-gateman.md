# Gateman Verification Report — BU-212 Durable Workflow Kernel

**Feature:** persisted-store-compatible fan-out/join workflow kernel
**Branch:** `codex/bu-212-durable-workflow`
**Date:** 2026-07-10
**Verifier:** Codex (Gateman analysis)

## Scope decision

This first runtime slice provides a deterministic workflow state machine with a persistence interface. It covers dependency validation, concurrent ready-step execution, retries, cancellation, budgets, event emission, and terminal run persistence. A production store, queue adapter, and real sandbox factory remain integration work.

## Sections 1–8

1. **Input validation — PASS.** IDs, workspace, positive budget, duplicate steps, and unknown dependencies are rejected.
2. **External identifiers — PASS.** Workspace identity is explicit and passed into each run; no provider or tenant defaults are invented.
3. **Side-channel/fraud surface — PASS with follow-up.** The kernel does not issue money or invoke tools; BU-207/209 policy must wrap actual agents.
4. **Configuration — PASS.** No environment reads or hardcoded service endpoints.
5. **Money/time/reversibility — PASS with follow-up.** Global budget, per-step retry count, AbortSignal cancellation, and idempotent `runId` input are represented; persistent resume semantics and per-step timeout enforcement belong to the store/worker adapter.
6. **Failure modes — PASS.** Failed prerequisites stop the graph; transient failures retry within the declared attempt budget; cycles fail loudly.
7. **Observability — PASS.** Every lifecycle transition is appended to the injected store and returned in the run timeline.
8. **AI output verification — PASS.** Tests prove fan-out/join, retries, and pre-aborted cancellation. No model output is trusted as a control-plane identifier.
