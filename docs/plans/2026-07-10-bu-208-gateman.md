# Gateman Verification Report — BU-208 Filesystem BUFI Roster

**Feature:** filesystem-first BUFI agent roster in `@open-agents/agent`
**Branch:** `codex/bu-208-filesystem-roster`
**Date:** 2026-07-10

## Scope decision

This slice adopts the Vercel Eve filesystem layout without importing Eve into the Open Agents runtime. Each agent owns `agent.ts`, `instructions.md`, `tools/index.ts`, and `workflow.ts`. The roster exports typed definitions and pure workflow plans. BU-212 owns durable execution; BU-207/BU-209 own real wallet-tool injection and policy.

## Sections 1–8

1. **Input validation — PASS.** Workflow planning rejects empty goals/workspace IDs and duplicate step IDs; unknown roster IDs fail explicitly.
2. **External identifiers — PASS.** Workspace ID is an explicit workflow input; no provider, tenant, wallet, or credential identifiers are invented or persisted by this layout-only slice.
3. **Side-channel/fraud surface — PASS.** Capability bundles are declarative least-privilege names. Eve exposes an intent capability, not a direct wallet executor; no money or external state is mutated.
4. **Configuration — PASS.** No environment reads, chain constants, provider URLs, or duplicated runtime configuration were added. The filesystem definitions are deterministic imports.
5. **Money/time/reversibility — N/A for this slice.** Workflow plans contain no amounts, timestamps, or state-changing calls; execution and approval semantics are explicitly deferred to BU-209/BU-212.
6. **Failure modes — PASS with follow-up.** Invalid roster IDs and malformed plan inputs fail loud. Durable retries, cancellation, budget, and sandbox isolation are BU-212 responsibilities, not hidden in these pure plans.
7. **Observability — N/A with follow-up.** This slice only builds static descriptors and plans; tracing belongs to BU-213 and durable execution to BU-212. No external work is silently performed.
8. **AI output verification — PASS.** Every new file was re-read; exports and imports were grep-verified; the baseline test failed before the roster existed; the local Eve reference was inspected and the implementation intentionally uses the layout rather than claiming a direct Eve runtime integration.
